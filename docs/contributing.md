# Contributing

Thank you for contributing to Ngawi.

## Development flow

1. Create a branch from `main`.
2. Implement a focused change.
3. Run full tests.
4. Open a pull request with clear notes.

## Local checklist

```bash
make clean
make debug
make test
```

## Commit style

Use clear prefixes:

- `feat(...)`
- `fix(...)`
- `refactor(...)`
- `test(...)`
- `docs(...)`

## Scope rules

- Keep each commit small.
- Add or update tests with feature changes.
- Update docs when syntax or behavior changes.

## Reporting issues

Include:

- Ngawi source sample
- command used
- current output
- expected output
