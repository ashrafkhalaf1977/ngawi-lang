# Roadmap

This roadmap tracks near-term language work.

## Next up

- cleaner diagnostics wording across parser and sema
- better parser recovery around nested blocks
- basic standard library growth

## Planned language features

- arrays
- structs
- `match` expansion to `bool` and `string`
- optional semicolon mode (evaluation)

## Longer term

- optimization pass strategy
- package management model
- richer tooling around formatter and linter

## Out of scope for current stage

- classes/OOP system
- generics
- VM or JIT backend
